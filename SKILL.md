# 大乐透标准分析流程 (daletou-analysis)

> **一句话**：把大乐透近 20 期 + 上期开奖数据，按固定 5 步流程做成「冷热清单 / 三区校验 / 后区搭配 / 选号约束 / 成品号码」的可复现分析报告；再叠加一个"最大覆盖优化"引擎，用蒙特卡洛诚实量化它能（和不能）做什么。

## 何时使用

- 用户说"跑大乐透""大乐透选号""大乐透分析""给我5组大乐透号码""大乐透推荐"。
- **默认严格按 5 步顺序执行**，不可遗漏或乱序。
- 如需"尽最大努力争取最大中奖可能性"，在 5 步法基础上加跑 `dlt_maxcover.py`（优化引擎）+ `dlt_review.py`（复查）。

## ⚠️ 诚实边界（每条产出都必须带，不可省略）

1. **一等奖概率 = 1 / 21,425,712（前区 C(35,5) × 后区 C(12,2)），由组合数决定，任何选号策略零影响**——这是铁律，任何数学模型都撼不动。
2. 冷热号、三区、胆码、连号全是**历史描述**，对未来开奖**零预测力**。每期开奖独立随机。
3. 本流程的价值在**数据归档 + 规律描述 + 娱乐仪式感**，不在于提高中奖率。
4. **理性购彩立场**：10 元、偶尔、无影响 = 娱乐消费非投资。返奖率约 51%，长期必亏；切勿因"多买总能中"心理滑向频繁大额投入。
5. 所有产出仅作演示，**绝不构成任何购彩建议**。

## 标准 5 步流程

| 步 | 名称 | 核心动作 | 脚本 | 主要产出 |
|---|---|---|---|---|
| 1 | 数据存档 | 近 20 期 + 上期原始数据，拆分前/后区，输出**冷热号码清单**（频次 ≥均值+0.5std 为热，≤均值−0.5std 为冷，附当前遗漏期数） | `analyze.py` | `dlt_raw_21.csv` `dlt_front_coldhot.csv` `dlt_back_coldhot.csv` |
| 2 | 前区分区规则 | 前区 1-10 / 11-20 / 21-35 三区，校验每期 5 码是否**三区全覆盖**（每区≥1），输出分区校验说明 + 历史满足率 | `zone_check.py` | `dlt_zone_check.csv` |
| 3 | 后区限定规则 | 划分后区冷热号，**两码和值锁定 9-16**，**禁止全冷/全热组合**，枚举合法搭配池 | `back_rule.py` | `dlt_back_valid_combo.csv` `dlt_back_hotcold.csv` |
| 4 | 选号参考约束 | 前区**连号**分析 + **23 区间胆码**（二区+三区，一区不纳入）+ **防扎堆单区**（任一区≥4 视为扎堆，须每区≤3），输出每组区间分布明细 | `select_ref.py` | `dlt_sel_ref.csv` `dlt_23_danma.csv` |
| 5 | 成品号码输出 | 生成 **5 组标准"5 前 2 后"** 号码，每组附**冷热配比 / 分区详情 / 备注**，统一汇总 | `final_numbers.py` | `dlt_final_groups.csv` |

> 第 1–4 步产出的 CSV 是单一数据源，第 5 步与优化引擎均从它们读取，保证口径一致。

## 口径铁律（跨步骤统一，自定义约束须显式声明）

- **前区三区**：一区 01-10 ｜ 二区 11-20 ｜ 三区 21-35（三区池 15 个，一/二区各 10 个）
- **后区冷热**（基于 21 期频次，均值 3.50 / 标准差 1.38）：热 [02,11,12] ｜ 冷 [03,06,09] ｜ 温 [01,04,05,07,08,10]
- **后区两码**：和值 ∈ [9,16]，禁止 冷冷 / 热热
- **23 区间** = 二区(11-20) + 三区(21-35)，共 25 码，胆码池；**一区(01-10)不纳入胆码池**
- **强胆**（23 区间 ≥4 次）：13 / 26 / 15 / 19 / 21 / 12 / 20 / 22 / 29
- **防扎堆**：5 码强制三区均衡，任一区 ≤3，严禁单区 ≥4

> 上述三区 / 和值 / 胆码 / 奇偶等**均为分析者自定义选号/过滤约束**；大乐透官方**无**此类规定。回测显示历史仅约 71%（三区）/ 52%（后区规则）满足，即这些约束会系统性排除一部分历史开奖形态——这是预期内的"过滤"，不是"预测"。

## 优化层：最大覆盖引擎（可选，第 6 层）

`dlt_maxcover.py` 在 5 步法硬约束之上，用**最大覆盖贪心算法**选 5 注，使 5 注在 35+12 的号码空间铺得最散（前区去重覆盖 /35、后区 /12），叠加 λ=0.03 历史热度权重。

**数学诚实性**：
- **可优化**：5 注越分散 → 同时不中的相关性越低 → 「至少命中任一阵容（含末等奖）」联合概率越高。蒙特卡洛验证约 **31% vs 随机 29%**（相对 +8%，绝对 <1pp）。
- **不可优化**：一等奖概率恒 1/2143 万，选号零影响。标准 5 步流程组本身已 30.75%（三区均衡天然分散），与优化版仅差 0.33pp——**日常用标准 5 组已足够**。

`dlt_review.py` 用更大样本（N=40 万）+ 95% 置信区间 + 两比例 z 检验 + 单注理论率验证，**复查并防止结论被噪声夸大**（曾纠正过一次"+2.44pp"虚高，真实仅 +0.74pp）。

## 使用方法

```bash
cd skills/daletou-analysis/scripts
python analyze.py        # 1 数据存档
python zone_check.py     # 2 前区分区校验
python back_rule.py      # 3 后区冷热搭配
python select_ref.py     # 4 选号参考约束
python final_numbers.py  # 5 成品号码（标准5组）

# 可选：优化 + 复查
python dlt_maxcover.py   # 最大覆盖优化5组
python dlt_review.py     # 蒙特卡洛复查
```

- 数据默认写入脚本同级 `../data/`（可用环境变量 `DLT_DATA_DIR` 覆盖，便于指定自己的开奖数据目录）。
- 所有脚本无第三方依赖，仅用 Python 标准库。

## 数据更新

内置 `analyze.py` 的 `RAW` 列表含默认 21 期（第 2026059–2026079 期，截至 2026-07-15）。出新一期后：
1. 抓取最新一期开奖号，按相同格式插入 `RAW` 列表最前（保持「最新在前」），并删掉最旧一期以维持 21 期窗口；
2. 重新跑 1→5 步（及可选优化），所有下游 CSV/HTML 自动刷新。

> 数据来源建议：中国体彩网官方公告 / 新浪彩票 / 各大彩票数据站交叉核对，确保单一权威源。
